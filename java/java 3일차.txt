30강 변수의 스코프와 static 

스코프 >> c언어에서 전역변수 / 지역변수 생각하면 됨 

모든 클래스는 인스턴스화 하지 않은 채로 사용할 수 없다

main에 static 이라는 키워드가 붙어있는데 
지금까지 main 메서드를 정의할 때 new를 사용하지 않아도 실행된 이유
static 이라는 키워드를 붙여주면 클래스를 인스턴스화 하지 않아도 static한 메서드나
static 한 변수를 사용할 수 있다.

static한 필드는 인스턴스를 생성할 때 만들어지는 것이 아니다. 
값을 저장할 수 잇는 공간도 하나밖에 없다
즉, 값을 공유한다 

31강 열거형 
열거형은 서로 관련 있는 상수들을 모아서 집합으로 정의한 것이다
열거형이 등장하기 전에는 public 과 static final을 이용해 상수를 열거형 대신 사용했다

EX)
package javaStudy;

public class EnumExam {

	public static final String MALE="MALE";
	public static final String FEMALE="FEMALE";
	
	public static void main(String[] args) {	

	}
}

static 하게 선언하면 객체 생성 없이 사용이 가능
final은 상수를 선언할 때 사용하는 키워드 

열거형 사용 방법
enum 열거형 이름 { 값1, 값2 }

***
Enum 이름은 클래스 명명 규칙을 따르며, 세부 항목에 이름은 모두 대문자를 사용한다

자바의 자료형은 크게 
기본자료형 ( int, float, char...)
참조자료형 (class,...) 으로 나뉜다
enum은 참조 자료형에 속하며 enum형을 가진 형태의 클래스이다

enum은 열거형이라는 아에 다른 데이터 타입이다
상수로 쓰던 것의 부족한 점을 해결하기 위해 만들어진 것이 enum이라고 이해하면 좋다

public class 클래스명 {
	자료형 필드명;

생성자 
특징 1 : 리턴형을 가지고 있지 않음
특징 2 : 프로그래머가 생성자를 만들지 않으면 컴파일할 때 매개변수가 없는 생성자가 
자동으로 생성된다. 매개변수가 없는 생성자를 기본 생성자라고 한다

33강 >> this
ex1)
public class Car{
	String name;
	int number;
}

	public Car( String name) {
	
	name = name;

	}

>> name = name; 때문에 이름이 같아서 두 name 모두 매개변수라고 생각하게 됨

34강 메서드 오버로딩

매개변수의 개수나 형(type)이 조금씩 다르지만 비슷한 기능을 하는 메서드가 여러개 있을 수 있다
이런 메서드를 각각 다른 이름으로 정의했다면 언제 어떤 메서드를 사용해야 할지 구분이 어렵다
따라서 매개변수의 수가 다르거나 형이 다른경우 같은 이름으로 메서드를 여러 개 정의할 수 있는데
이를 메서드 오버로딩이라고 한다

35강 >>생성자 오버로딩과 this
생성자도 메서드와 마찬가지로 여러 개 선언할 수 있다
매개변수의 수와 형만 다르다면 생성자를 여러 개 선언할 수 있다

Car 객체에 생성자가 하나도 없을 때는 기본 생성자가 자동으로 만들어지지만, 
생성자를 하나라도 정의하면 기본 생성자가 자동으로 만들어지지 않는다

이렇게 기본 생성자로도 Car객체를 만들고 싶다면 기본 생성자를 직접 정의해두면 된다
ex)
public Car( ) {

}

name 과 number를 받는 생성자도 정의
ex)
public Car (String name, int number) {
	this.name = name;
	this.number = number;
}
